## الهدف
نخلّي البوت يتعامل مع كل زبون كأنه نادل يعرفه شخصياً منذ سنين — يتذكر طلباته، تفضيلاته، يقترح عليه بذكاء، ويسرّع التجربة بدل ما يعيد نفس الأسئلة كل مرة.

## الوضع الحالي (شنو موجود فعلاً)
- جدول `customer_memory` فيه: اسم، عدد طلبات، LTV، آخر عنوان/هاتف، تفضيلات نصية، ملاحظات.
- أداة `recall_customer` ترجع هذي المعلومات.
- أداة `suggest_upsell` تقترح مكمّل بعد add_to_cart.
- الميزة كلها **opt-in** خلف feature flag، والتفضيلات تنكتب يدوياً من المالك فقط.

## المشاكل الجوهرية
1. الذاكرة ما تتذكر **شنو طلب الزبون فعلاً** — بس عدد الطلبات والـ LTV. ما يكدر يقول "نفس طلبك السابق؟".
2. التفضيلات (بدون مايونيز، حار، نباتي) ما تنستخرج تلقائياً من المحادثات — لازم المالك يكتبها يدوياً، وهذا ما يصير.
3. ما اكو طريقة سريعة لإعادة الطلب — لازم الزبون يطلب كل صنف من جديد.
4. الميزة مغلقة افتراضياً ومخفية خلف flag — يعني أغلب المطاعم ما يستفيدون منها.
5. المالك ما يشوف بروفايل الزبون داخل لوحة المحادثات — يعني ما عنده context لما يستلم محادثة.

## الخطة (5 تحسينات متكاملة)

### 1. ذاكرة طلبات حقيقية (Last Orders + Favorites)
- توسيع `recall_customer` ليرجّع:
  - **آخر 3 طلبات** بأصنافها وأسعارها وتواريخها.
  - **أكثر 5 أصناف طلباً** لهذا الزبون (favorites).
- مصدر البيانات: جدول `orders` الموجود (نقرأ items JSON ونجمّعها).

### 2. أداة جديدة: `reorder_last`
- البوت يقدر يضيف آخر طلب كامل للسلة بضغطة واحدة من جانبه.
- الزبون يكتب "نفس الطلب" / "زي المرة الماضية" → البوت ينفذ مباشرة بدون أسئلة.
- يتحقق من توفر الأصناف الحالي (is_available + stock).

### 3. استخراج التفضيلات تلقائياً (Auto-learn)
- بعد كل طلب مؤكد، تشتغل دالة خلفية (server function) ترسل تاريخ المحادثة + الطلب لـ Lovable AI (`google/gemini-2.5-flash-lite` — رخيص وسريع)، وتطلب منه يستخرج:
  - تفضيلات ثابتة ("ما يحب البصل"، "يفضل الحار"، "نباتي").
  - حساسيات/تحذيرات ("حساسية مكسرات").
- النتيجة تتدمج مع `preferences` الموجود (merge مو overwrite).
- يشتغل عبر database trigger على `orders` عند status='confirmed'.

### 4. الذاكرة افتراضياً مفعّلة + استدعاء تلقائي
- إزالة feature flag — الذاكرة تشتغل لكل المطاعم.
- بدل ما نعتمد على الموديل ينادي `recall_customer`، نحقن النتيجة **مباشرة في system prompt** أول كل محادثة (لو الزبون قديم).
- توفير: استدعاء tool أقل = أسرع وأرخص.
- الموديل يبلش المحادثة وهو يعرف الزبون بدون round-trip.

### 5. بروفايل الزبون في لوحة المحادثات
- في `dashboard.tsx` فوق المحادثة المفتوحة، نضيف بطاقة جانبية صغيرة:
  - اسم الزبون + عدد طلباته + LTV.
  - آخر 3 طلبات (قابلة للنقر لفتح تفاصيل الطلب).
  - تفضيلات مستخرجة + ملاحظات المالك (قابلة للتعديل).
- المالك يشوف الـ context فوراً لما ياخذ handoff.

## التفاصيل التقنية

### Migration
```sql
-- توسيع recall_customer ليرجّع آخر الطلبات والمفضّلات
CREATE OR REPLACE FUNCTION public.recall_customer(_conversation_id uuid)
RETURNS jsonb ...
-- يضيف:
--   'recent_orders': آخر 3 من orders (items, total, created_at)
--   'favorites': top 5 menu_items من orders تاريخياً
--   'memory_id': للاستخدام في update من الـ trigger

-- جدول جديد للتحذيرات (منفصل عن preferences النصية)
ALTER TABLE customer_memory ADD COLUMN auto_preferences jsonb DEFAULT '{}'::jsonb;
-- شكل: {"dislikes": ["بصل","مايونيز"], "likes": ["حار"], "allergies": ["مكسرات"], "diet": "نباتي"}

-- Trigger يستدعي edge function بعد تأكيد الطلب
-- (أو نستخدم pg_net للاستدعاء، أو نخليه inline في agent-run بعد submit_order)
```

### تعديلات `supabase/functions/agent-run/index.ts`
- إزالة شرط `memoryEnabled` — الميزة دائماً مفعّلة.
- بعد جلب conv، نستدعي `recall_customer` مباشرة في الكود (مرة واحدة)، ونحقن النتيجة في system prompt كقسم "ملف الزبون".
- إزالة tool `recall_customer` من القائمة (ما عاد ضروري — أول call تلقائي).
- إضافة tool `reorder_last` مع handler يفحص availability.
- بعد كل `submit_order` ناجح، نطلق استخراج التفضيلات (fire-and-forget).

### دالة استخراج التفضيلات
- ملف جديد: `supabase/functions/extract-preferences/index.ts`.
- input: conversation_id + order_id.
- يقرأ آخر 20 رسالة + items الطلب → يرسلها لـ Lovable AI مع tool calling لاستخراج JSON منظم.
- يدمج النتيجة مع `auto_preferences` الموجودة (union للقوائم).

### تعديلات `dashboard.tsx`
- مكوّن جديد `<CustomerProfileCard conv={selectedConv} />` يظهر في sidebar.
- يستدعي `recall_customer` عبر RPC ويعرض النتيجة.
- editable: ملاحظات + تفضيلات يدوية.

> بوت: ممكن توضح شنو تحب؟

> زبون: إي
> بوت: تمام، نأكّد؟ → [زر تأكيد]

- مكوّن جديد `CustomerProfileCard` في dashboard.
