# خطة: نطاق توصيل جغرافي دقيق + كشف لغة الزبون

الهدف: كل فرع يكون له **حدود توصيل جغرافية حقيقية** (مو بس أسماء مناطق نصية)، والوكيل يفحص موقع الزبون عليها قبل ما يأكد التوصيل. وبنفس الوقت يتعرف على لغة الزبون (عربي/كردي/تركماني/إنجليزي) ويرد بنفس اللغة.

## شنو راح يصير عملياً

### بإعدادات الفرع (Dashboard → الفروع → تعديل)
يضيف المالك حقل جديد اسمه **"نطاق التوصيل الجغرافي"** فيه ثلاث خيارات (مالك يختار لكل فرع على حدة):

1. **محافظة كاملة** — قائمة منسدلة بالـ 18 محافظة عراقية. النظام عنده حدود المحافظات جاهزة (polygons محفوظة) فما يحتاج المالك يرسم شي.
2. **رسم منطقة على الخريطة** — خريطة تفاعلية (Google Maps) يرسم عليها المالك حدود التوصيل بنقرات (polygon). يقدر يعدل/يحذف نقاط.
3. **دائرة حول الفرع** — يحدد نصف القطر بالكيلومترات (مثلاً 8 كم) ويعتمد على موقع الفرع الحالي.

ويبقى حقل `delivery_areas` النصي الموجود حالياً كـ **مساعد إضافي** (للزبون اللي يرسل عنوان نصي بدون موقع GPS).

### بمحادثة الوكيل
سيناريو 1 — الزبون يرسل موقع GPS (location pin من تيليجرام/واتساب):
- الوكيل يفحص الإحداثيات على نطاق الفرع → يقبل أو يرفض فوراً.

سيناريو 2 — الزبون يكتب عنوان نصي:
- الوكيل يحاول يطابق العنوان بـ `delivery_areas` (الموجود حالياً).
- إذا ما تطابق → **يطلب من الزبون يرسل موقع GPS** (رسالة "ممكن ترسل موقعك على الخريطة؟").
- إذا أرسل الموقع → يفحص النطاق الجغرافي.

سيناريو 3 — الزبون خارج النطاق (حسب طلبك):
1. سؤال توضيحي واحد لو المنطقة مبهمة.
2. لو تأكد إنه خارج التغطية:
   - اعتذار مهذب.
   - لو فرع ثاني يغطي → يقترحه.
   - يعرض خيار **الاستلام من المطعم (Pickup)**.
   - يسجل العنوان بجدول **"مناطق مطلوبة غير مغطاة"** (`uncovered_requests`) تطلع للمالك بصفحة بلوحة التحكم.
3. ما يقبل الطلب توصيل إلا لو الموظف وافق يدوياً (زر "تأكيد استثنائي" بلوحة التحكم).

### كشف اللغة (الوكيل)
- يكشف لغة الزبون من **نص رسالته** (عربي، كردي صوراني/بادينان، تركماني، إنجليزي).
- يرد بنفس اللغة طوال المحادثة، حتى لو تغيرت بمنتصف الكلام.
- اسم الصنف بالمنيو يبقى زي ما هو (مو راح نترجم المنيو تلقائياً)، بس باقي الكلام (التحية، الأسعار، تأكيد الطلب، رسالة الرفض) كلها بلغة الزبون.
- إذا اللغة غير معروفة (مثلاً رموز فقط) → يرجع للغة الافتراضية للمطعم.

## ما اللي **ما** راح يتغير
- البوتات والربط مع تيليجرام/واتساب نفس الشي.
- جدول المنيو نفسه (ما نضيف ترجمات).
- منطق العربة والـ preview/submit.
- لوحة التحكم الحالية (نضيف بس تبويب صغير + خريطة بشاشة تعديل الفرع).

## معايير القبول
- مالك يقدر يختار محافظة كاملة من قائمة → الوكيل يرفض أي GPS خارج حدودها.
- مالك يقدر يرسم polygon على الخريطة → الفحص يشتغل على نفس الـ polygon.
- زبون يرسل عنوان نصي مبهم → الوكيل يطلب موقع GPS مرة وحدة.
- زبون كردي يبدأ بـ "سلاو" → الوكيل يكمل كردي.
- زبون خارج التغطية → اعتذار + اقتراح Pickup + تسجيل بجدول الطلبات غير المغطاة.
- صفحة جديدة بلوحة التحكم: **"مناطق مطلوبة"** تبين العناوين المرفوضة مع عدد التكرار، عشان المالك يقرر يوسع.

## التفاصيل التقنية

**جداول/أعمدة جديدة:**
- `branches.coverage_type` text: `'governorate' | 'polygon' | 'radius'`.
- `branches.coverage_governorate` text (كود المحافظة، nullable).
- `branches.coverage_polygon` jsonb (مصفوفة [{lat, lng}]، nullable).
- `branches.coverage_radius_km` numeric (nullable).
- `branches.languages` text[] (default `['ar']`) — للسجل فقط، الكشف يصير من النص.
- جدول جديد `uncovered_requests`: `restaurant_id, branch_id?, conversation_id, customer_phone, address_text, lat, lng, count, first_seen_at, last_seen_at`. مع GRANT + RLS عبر `restaurant_id`.
- بيانات ثابتة: ملف JSON بالـ repo فيه polygons المحافظات الـ 18 (نجيبها من مصدر مفتوح مثل OSM/GeoBoundaries).

**ملفات تتعدّل:**
- `src/components/branches-tab.tsx` — UI جديد للنطاق + خريطة Google Maps لرسم polygon (نستخدم Google Maps connector — مفعّل أصلاً).
- `src/lib/geo.ts` (جديد) — دوال `pointInPolygon`, `pointInRadius`, `pointInGovernorate`.
- `supabase/functions/agent-run/index.ts` — أداة جديدة `check_coverage(lat,lng)` + توسيع `set_delivery_info` لقبول `customer_location {lat,lng}` + منطق الـ fallback لطلب الموقع + اقتراح Pickup + تسجيل بـ `uncovered_requests`. كشف اللغة عبر heuristic (Unicode ranges + كلمات مفتاحية) ثم حقن بالـ system prompt.
- `supabase/functions/telegram-webhook/index.ts` — استقبال رسائل `location` من تيليجرام وتمريرها بـ `meta.customer_location`.
- صفحة جديدة `src/routes/_authenticated/uncovered.tsx` (أو تبويب داخل dashboard) — قائمة المناطق المطلوبة مع زر "تأكيد استثنائي".
- `src/components/dashboard-page.tsx` — إضافة التبويب الجديد.

**Connector:** Google Maps (موجود — `VITE_LOVABLE_CONNECTOR_GOOGLE_MAPS_BROWSER_KEY`) للخريطة التفاعلية برسم الـ polygon.

**ما يتغير بـ schema:** ما نلمس `auth`, `storage`, ولا نعدل الموديل/الـ tools الأخرى للوكيل.
